<description>
Writes a file to the local filesystem. Overwrites existing files.
</description>

<critical-rules>
<rule>If modifying an existing file, you MUST use the Edit tool instead. The Write tool is ONLY for creating genuinely new files that do not exist yet.</rule>
<rule>Before using Write, you MUST use Read to check if the file already exists. If it does, use Edit instead.</rule>
<rule>NEVER use Write to recreate an existing file with modifications. This destroys version history and is strictly forbidden.</rule>
<rule>NEVER proactively create documentation files (*.md), README files, summary files, or guide files unless the user explicitly requested it.</rule>
<rule>Only use emojis if the user explicitly requests it.</rule>
</critical-rules>

<when-to-use>
Use Write ONLY when ALL of these conditions are true:
1. The file does not currently exist
2. The user's task genuinely requires a new file (e.g., creating a new component, test file, or config)
3. You have verified the file doesn't exist by checking the filesystem
</when-to-use>
