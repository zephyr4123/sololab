在单次操作中对同一文件进行多次编辑的工具。它基于 Edit 工具构建，允许你高效执行多个查找替换操作。当你需要对同一文件进行多次编辑时，优先使用此工具而不是 Edit 工具。

使用前：
1. 使用 Read 工具了解文件内容和上下文
2. 验证目录路径正确

提供以下内容进行多文件编辑：
1. file_path：要修改的文件的绝对路径
2. edits：要执行的编辑操作数组，每个编辑包含：
   - oldString：要替换的文本（必须与文件内容完全匹配，包括所有空白和缩进）
   - newString：替换 oldString 的编辑文本
   - replaceAll：替换所有出现的 oldString。此参数可选，默认为 false

重要：
- 所有编辑按提供的顺序依次应用
- 每个编辑在前一个编辑的结果上操作
- 所有编辑必须有效才能成功——如果任何编辑失败，全部不会应用（原子操作）
- 仔细规划编辑以避免顺序操作之间的冲突
- 确保所有编辑生成符合规范的、正确的代码
- 始终使用绝对文件路径
- 仅在用户明确要求时使用 emoji
- 使用 replaceAll 在整个文件中替换和重命名字符串
